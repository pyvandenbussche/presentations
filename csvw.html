Tabular Data and CSV

  • Tabular data is structured into rows
  • Rows are divided into the same number of cells corresponding to the columns

"country","country group","name (en)"
"at","eu","Austria"
"be","eu","Belgium"
country code country group name (en)
at eu Austria
be eu Belgium
  • In CSV, a separator character is used to delimit cells (comma, semicolon, tab)
  • The header row can be used to define columns titles

Tradeoff simplicity/expressivity: CSV model does not allow schema information such as type, value range, datatype, language, etc.

CSVW

  • W3C CSV on the Web Working Group
  • Recommendations published on the 17 December 2015
  • Covers comma/semicolon/tab-delimited files as well as fixed field formats, spreadsheets, HTML tables, and SQL dumps as tabular data files
  • Defines a metadata vocabulary for tabular data
  • Specifes a model for tabular data and metadata along with the way to publish it on the Web
  • Transformations CSV2JSON and CSV2RDF are defined
